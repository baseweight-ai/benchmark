# Baseweight — Agent Instructions

You are working in the Baseweight benchmark pipeline. Your job is to make changes
that are correct, reproducible, and aligned with the principles below. Quality
bar is research-grade, not demo-grade.

## Read before every change

- `principles/ml-pipeline-principles.md` — modularity, reproducibility, data mgmt, observability, orchestration, eval, artifacts
- `principles/ml-rigor-principles.md` — training/convergence, data quality, metric design, experimental design, numerical correctness, reporting

Before editing, identify which principle(s) your change likely impacts and re-read
those.

## Hard rules

- Never tune on test data, even indirectly.
- No silent error swallowing — errors get stage + input context.
- Never hand-edit pipeline outputs (`results/`, `runs/`, `dashboard-data/` or `workspace/checkpoints/`). Can only be regenerated by running the pipeline.
- Never hand-edit data under `data/`. Treat it as read-only.

## Workflow

1. Name the principle(s) you're touching (e.g. "reproducibility", "metric design").
2. Make the smallest change that works.
3. Run all pytests before claiming it works.

## Always stop and ask before

- Bumping pinned dependency versions
- Adding a new external API dependency
